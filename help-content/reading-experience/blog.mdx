# Content Folders

Verto does not have a separate blog subsystem. Any folder in the configured
content source becomes part of the same Library and Reader. A `blog/` folder is
one useful convention, but it uses the same navigation, typography, search,
tags, and document actions as every other folder.

## Article Frontmatter

Frontmatter is optional. Add only the fields the article needs:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | No | Display title; falls back to the first heading or filename |
| `description` | string | No | Short summary used in document lists and metadata |
| `date` | string | No | Publication date in `"YYYY-MM-DD"` format |
| `author` | string | No | Author name shown in the document masthead |
| `tags` | string[] | No | Labels linked to the matching tag view |
| `order` | number | No | Optional ordering value for sidebar placement |

A complete frontmatter block looks like this:

```yaml
---
title: "Building a Custom Remark Plugin"
description: "How we built Verto's inline comment system from scratch."
date: "2026-02-15"
author: "tsaiggo"
tags: ["remark", "mdx", "plugins"]
order: 1
---
```

## Creating a Post

Three steps:

1. **Create the file.** Add a new `.mdx` or `.md` file in `content/blog/`. The filename becomes the URL slug, so `content/blog/my-first-post.mdx` becomes `/read/blog/my-first-post`.

2. **Add frontmatter if useful.** Title, description, date, author, tags, and order are all optional.

3. **Write your content.** Everything below the frontmatter is standard MDX. You can use the same block components available in any Library or Help document.

```mdx
---
title: "My First Post"
description: "Getting started with Verto's blog."
date: "2026-03-01"
author: "tsaiggo"
tags: ["getting-started"]
---

# My First Post

Write your content here. All MDX features and block components work just like in docs.
```

## Folder Index Page

The `/read/blog` route is the folder index. If the folder has no index file,
Verto generates a directory listing from its readable documents and nested
folders. Add `README.md`, `index.md`, or `_index.md` when the folder needs
an authored introduction.

Use `order` and `content/navigation.json` when you need deterministic
navigation. Use date or updated metadata for the Recently updated view.

## Tag Display

Tags appear in the document masthead and link to `/read/tags/{tag}`. The
top-level Tags page also summarizes tags across the active Library.

## URL Structure

Folders follow the normal Reader URL scheme:

| URL | Page |
|-----|------|
| `/read/blog` | Generated or authored folder index |
| `/read/blog/{slug}` | Article whose slug matches the filename |

The slug comes directly from the source path. There is no separate magazine
card grid or hidden publishing model to maintain.
