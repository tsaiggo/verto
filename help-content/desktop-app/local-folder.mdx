# Open a Local Folder

Open **Sources → Local Library** to turn a folder of `.md` and `.mdx` files
into the live Library. Verto uses a folder picker in both the desktop app and
supported browsers; the read-only Folder field reports what was selected.

## Browser and desktop behavior

| Capability | Supported browser | Verto desktop |
|------------|-------------------|---------------|
| Choose a directory | Browser directory picker, with a directory-upload fallback | Native operating-system folder picker |
| Read files | Cached copies stored on this device | Files read directly from the authorized folder |
| Refresh changed files | Choose the folder again | Reconnect or reopen the authorized folder |
| Save edits back to that folder | No; the web Editor exports a file | Yes; the Editor can write through native commands |
| Workspace state | Browser storage on this device | Portable `.verto/` state beside the library |

The browser does have directory selection. Verto prefers
`showDirectoryPicker()` when available and falls back to a hidden directory
file input. It recursively reads the selected folder, then stores readable file
text in IndexedDB and the small folder index in local storage.

<Callout type="warning">
  A browser connection is a device-local cache, not a live filesystem mount.
  Verto does not retain general write access to the directory. Choose the
  folder again to import changes made outside Verto.
</Callout>

If directory selection is unavailable in the current environment, Verto shows
that folder access is unavailable. It does not accept a manually typed path,
because a web page cannot gain permission from path text alone.

## Choose and connect

In the browser, **Choose and connect** selects the directory, caches readable
files, and immediately makes that selection the active Library.

In the desktop app:

1. Select **Choose folder** and approve a directory in the native picker.
2. Review the selected path.
3. Select **Connect library** to make it active.

Connecting replaces the live Library view; it does not delete or modify the
previous folder. Library and Explorer refresh after a successful connection.

## Folder feedback

Verto scans recursively, skips dotfiles and dot-directories, and reports one of
these states after selection or connection:

- **“Found _N_ readable files.”** — the folder contains that many Markdown or
  MDX files.
- **“No .mdx or .md files found in this folder.”** — the directory is valid but
  has no readable documents.
- **“That folder does not exist.”** or **“That path is a file, not a folder.”**
  — a previously authorized desktop path is no longer a usable directory.

Only `**/*.{mdx,md}` is read. Other file types stay untouched.

## Recent folders

Recent folder names are stored on this device.

- The desktop app retains native authorization for folders it has opened, so a
  recent folder can normally reconnect without another picker prompt.
- Browsers do not let history bypass directory permission. Verto keeps one
  cached browser selection; after choosing a different folder, choose an older
  one again before reconnecting it.

Disconnecting a local library stops using it. It does not delete source files,
recent-folder history, the current browser cache, or portable `.verto/` state.

## Runtime connection versus build source

A folder connected here is a **runtime local library**. It updates Library and
Explorer without rebuilding the application, and its documents open in the
runtime Reader.

This is separate from the local source used to pre-render `/read/*` during a
web or desktop build:

```bash
VERTO_CONTENT_SOURCE=local
VERTO_LOCAL_DIR=/path/to/build-library
npm run build
```

Those environment variables are for operators creating a build. You do not
need them to open a folder from Sources.
