# Environment Variables

Verto reads Markdown and MDX content from one configured source. The desktop
product uses Local Library and RSS subscriptions. Environment variables select
the build-time content source and enable the optional assistant.

<Callout type="info">
The configured content source is read at build time. Change a source variable
and rebuild Verto before expecting the library to update.
</Callout>

## Content source

| Variable | Default | Notes |
|----------|---------|-------|
| VERTO_CONTENT_SOURCE | local | local or onedrive |

### Local Library

| Variable | Default | Notes |
|----------|---------|-------|
| VERTO_LOCAL_DIR | content | Folder to read Markdown and MDX from; absolute or project-relative |

    VERTO_CONTENT_SOURCE=local
    VERTO_LOCAL_DIR=content

In the desktop app, choose a folder from Sources instead of editing the
variable. Verto scans nested folders and lists readable files in Library.

### OneDrive

OneDrive remains available as a build-time source.

| Variable | Default | Notes |
|----------|---------|-------|
| VERTO_ONEDRIVE_SHARE_URL | none | Public share link to the folder |
| VERTO_ONEDRIVE_PATH | none | Optional sub-folder inside the shared item |
| VERTO_ONEDRIVE_TENANT | common | Tenant, consumers, or a tenant GUID |
| VERTO_ONEDRIVE_CLIENT_ID | none | Entra application client id |
| VERTO_ONEDRIVE_CLIENT_SECRET | none | Entra application client secret |
| VERTO_ONEDRIVE_REFRESH_TOKEN | none | Refresh token for private content |

    VERTO_CONTENT_SOURCE=onedrive
    VERTO_ONEDRIVE_SHARE_URL=https://1drv.ms/u/s!...
    VERTO_ONEDRIVE_PATH=content

Remote sources do not reliably expose a per-file modification time. Prefer
frontmatter date, updated, or order values for deterministic sorting.

## AI Assistant

The assistant is off by default. The current backend uses GitHub Models.

| Variable | Default | Notes |
|----------|---------|-------|
| NEXT_PUBLIC_VERTO_ASSISTANT | none | github, copilot, or github-models |
| NEXT_PUBLIC_VERTO_ASSISTANT_MODEL | openai/gpt-4o-mini | Optional model override |

    NEXT_PUBLIC_VERTO_ASSISTANT=github
    NEXT_PUBLIC_VERTO_ASSISTANT_MODEL=openai/gpt-4o-mini

Verto does not include GitHub sign-in. Add a GitHub Models token manually in
Settings > AI & Agent. The token stays in localStorage on the current device
and is sent only to the configured inference endpoint.
