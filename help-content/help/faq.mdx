# FAQ

Short answers to the questions that come up most often. For step-by-step fixes
to things that aren't working, see [Troubleshooting](/help/help/troubleshooting).

<AccordionGroup>
  <Accordion title="Does Verto need a database?">
    No. Markdown and MDX files are the Library's source of truth. Static sources
    are indexed and rendered at build time; a connected local folder is indexed
    on the current device. Verto does keep workspace state: desktop libraries
    use a hidden `.verto/` directory, while web-only sessions use
    `localStorage`.
  </Accordion>
  <Accordion title="Can I use plain Markdown (.md), or is MDX required?">
    Both work. `.md` uses the same reading pipeline. Reach for `.mdx` only
    when a document needs JSX components such as Callout, Toggle, or Figure.
  </Accordion>
  <Accordion title="Which content sources are supported?">
    At runtime, Verto can connect a local Markdown/MDX folder to Library and can
    collect RSS or Atom feeds in Inbox. Static deployments can build from a
    local directory or OneDrive. RSS is not a Library replacement, and
    OneDrive is not a live desktop sync provider.
  </Accordion>
  <Accordion title="When do source changes appear?">
    A connected local folder is scanned on the current device, so its readable
    files can be reopened without rebuilding the application. Build-time local
    or OneDrive sources require a new build before remote changes appear.
  </Accordion>
  <Accordion title="What happens if a document uses a component Verto doesn't know?">
    It won't crash the whole page. Unmapped JSX tags render an
    `UnknownComponent` placeholder so the rest of the document stays
    readable. The placeholder names the tag Verto could not resolve.
  </Accordion>
  <Accordion title="How do I control the order of pages?">
    Add an `order` to frontmatter — lower numbers come first. Pages without an
    `order` fall back to date (newer first) and then title. You can also use
    [navigation.json](/help/reference/navigation-json) to rename, reorder, or
    hide entries without changing document frontmatter.
  </Accordion>
  <Accordion title="How do I hide a page?">
    Set `hidden: true` to hide it in every build, or `draft: true` to keep it
    visible in development and hide it in production. See
    [Frontmatter](/help/reference/frontmatter).
  </Accordion>
  <Accordion title="Do inline comments break in other Markdown tools?">
    No content is lost. The `[^c-N]` syntax falls back to an ordinary footnote
    in renderers that do not understand Verto's inline-comment convention. In
    Verto it becomes a click-to-reveal author note. See
    [Inline Comments](/help/core-concepts/inline-comments).
  </Accordion>
  <Accordion title="Is the Agent required?">
    No. Reading, search, writing, bookmarks, collections, tags, and Inbox work
    without AI. The supported live provider is GitHub Models. A build must
    enable that provider, and the current device must have a token in Settings.
    Verto does not sign you into GitHub. Development builds may instead expose
    deterministic demo responses.
  </Accordion>
  <Accordion title="What context does the Agent receive?">
    Verto sends a request only after you ask. Reader requests disclose whether
    they attach the full page or a normalized prefix. Workspace Agent requests
    likewise report when only part of a static source library was attached;
    they do not imply that unattached files were searched.
  </Accordion>
  <Accordion title="What is different in the desktop app?">
    The Tauri desktop app can choose a local folder, save edits back to it, and
    keep portable workspace state in that library's `.verto/` directory. The
    web editor exports an `.mdx` file instead of writing directly to disk, and
    browser folder access depends on browser support.
  </Accordion>
  <Accordion title="Where do Library and Help pages live?">
    Library documents use `/read/*`. Bundled product documentation uses
    `/help/*` and always comes from `help-content/`, so switching the active
    Library does not replace Help.
  </Accordion>
</AccordionGroup>
