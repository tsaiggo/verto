# Installation

Verto is currently distributed from its application repository, not as a
package on the public npm registry. Run it from an authorized checkout.

## Prerequisites

- **Node.js 20.9 or newer** (the minimum supported by this Next.js 16 build)
- npm, which is included with Node.js
- access to the Verto repository

<Callout type="warning">
  `npm install verto` does not install this application. The repository is
  private and the `verto` package is not published for consumer installation.
  Clone an authorized checkout, then install that checkout's dependencies.
</Callout>

## Run the web app

### 1. Clone the repository

```bash
git clone https://github.com/tsaiggo/verto.git
cd verto
```

The clone requires a GitHub account that can access the repository.

### 2. Install dependencies

```bash
npm install
```

### 3. Start development

```bash
npm run dev
```

Open `http://localhost:3000`. The default build-time Library comes from the
bundled `content/` folder, and Help always comes from `help-content/`.

The main source directories are:

```text
verto/
├── app/             # Next.js routes and product pages
├── components/      # Workspace, Reader, MDX, and form components
├── content/         # Default Library documents
├── help-content/    # Bundled product Help
├── lib/             # Content sources, MDX pipeline, state, and AI tools
└── src-tauri/       # Optional desktop shell and native commands
```

## Add or connect content

For a quick source-controlled document, add a `.md` or `.mdx` file anywhere
under `content/`:

```mdx
---
title: "My First Document"
description: "A portable note read by Verto."
tags: ["getting-started"]
---

# My First Document

Write in Markdown or use Verto's registered MDX components.
```

Its path becomes its Reader URL. For example,
`content/notes/my-first-document.mdx` becomes
`/read/notes/my-first-document`.

You can also open **Sources → Local Library** and choose a folder at runtime.
Supported browsers cache the selected Markdown and MDX files on this device;
the desktop app reads an authorized folder directly. Runtime local libraries
do not require a rebuild. See [Open a Local Folder](/help/desktop-app/local-folder)
for the browser and desktop differences.

## Production builds

Create a normal Next.js web build with:

```bash
npm run build
npm start
```

The selected build-time source is pre-rendered into `.next/`. Set build-source
environment variables before `npm run build`; changing that source requires a
new build. A standard Next.js host such as Vercel can run this output.

The Tauri desktop build uses a separate static export with no Node server at
runtime:

```bash
npm run build:tauri
```

Building an installer also requires the platform's Rust and Tauri toolchain.
Use `npm run tauri:dev` for desktop development and
`npm run tauri:build:unsigned` only for a local smoke-test installer.
