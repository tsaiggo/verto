# Authoring Experience

Verto reads each file's frontmatter and uses it to enrich the reading experience without you needing to touch any code.

## Frontmatter Fields

| Field | Effect |
|-------|--------|
| `title` | Document title (overrides first H1 / filename) |
| `description` | Used in metadata, previews, and the directory index |
| `date` | ISO date — shown in the header, drives sort for blog-style content |
| `author` | Shown next to the date |
| `tags` | Rendered as colored chips in the header; each chip links to a tag index |
| `cover` | Path or URL to a cover image rendered above the title |
| `draft` | When `true`, the document is hidden from production builds (visible in `next dev`) |
| `updated` | ISO date — overrides the file mtime in the "Updated …" footer |
| `lang` | Sets `<article lang="…">` for hyphenation and font fallbacks |
| `toc` | `false` to hide the table of contents, or `{ minDepth, maxDepth }` to control depth |
| `order` | Numeric sort hint for navigation |
| `hidden` | When `true`, omits the document from navigation |

## Table of Contents

By default Verto extracts h2 and h3. Set `toc.maxDepth: 4` in frontmatter to also include h4.

### Subsection (h3)

This appears in the TOC.

#### Sub-subsection (h4)

This *also* appears in the TOC because this page set `maxDepth: 4`.

To hide the TOC entirely, set `toc: false`.

## Footnotes

Verto renders standard CommonMark footnotes with a tidy bottom-of-page section[^1]. The `[^c-id]` variant becomes an inline-comment popup instead — see the [Inline Comments doc](/read/docs/core-concepts/inline-comments).

[^1]: Footnotes are powered by `remark-gfm`; backreference arrows return you to the source location.

## Cover Image

Set `cover: "https://…"` (or a relative path) and Verto renders a 16:7 banner above the title. Bring your own asset pipeline.

## Drafts

Mark a work-in-progress page with `draft: true`. In `next dev` it shows up with a yellow **Draft** pill so you can preview it; in production builds it's omitted from navigation entirely.

## Tag Index

Click any tag chip to land on `/read/tags/<name>`, an auto-generated index of every document carrying that tag.
